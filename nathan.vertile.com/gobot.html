architecture

language

  • formalized interface to a computer's functionality
  • e.g. JavaScript
- To talk about web technologies and frameworks, first we have to have a shared understanding of computer languages. - It let's you tell the computer what to do. - Language is not a limitation, since it is only a way to express a computer's capabilities.

program

  • written in a programming language
  • composed of a set of instructions
  • e.g. helloworld 
    console.log("hello world");

instruction

  • control structure 
    if( somethingIsTrue ) {
  // do something
}
  • read or write data 
    var name = "Nathan";
  • do something nice: comment, function, object etc.
- Comments are meant to clarify code. Functions and objects, from which functional and object oriented programming derive their names provide nice ways of changing the control flow or mutating data.

library

  • a collection of instructions that allow a user to perform a set of standard operations
  • e.g. jQuery 
    $('body').css({opacity: '.5', 'background-color': 'green'});

framework

  • a subset of or all of the components needed to create a web app, from the operating system to the web interface
  • e.g. Ruby On Rails, Laravel, Django

stack

  • a set of frameworks, libraries, idioms and best practices to solve a particular class of problem
  • technologies run on the server
  • may or may not include a framework
  • e.g. LAMP

protocol

  • http
  • request/response
  • client/server
  • osi layer 7
  • usually on tcp transport
- All webapps speak HTTP. - To be called a webserver or web client it is sufficent that the program communicate in HTTP.

main.js

$(function(){
  var ui = new Ui($('.game-container'), $('.message-container'));
  var game = null;
  var socket = io.connect();

  socket.on("connect", function(){
    console.log('connected');
    socket.emit('command', 'set_name', 'Nathan');
  });
  socket.on("gamestate", function(state){
    var GameState = JSON.parse(state);
    ui.resizeContainer(GameState);
    if (!game) {
      game = new Game(GameState.Constants, ui.getGameContainer(), $('.score-container'), socket);
    }
    game.draw(GameState);
    game.score(GameState);
  });
  socket.on("disconnect", function() {
    console.log("disconnected");
  });
  socket.on("gameevent", function(type, info){
    var json = JSON.parse(info);
    console.log("gameevent", type, JSON.stringify(json));
    switch (type) {
      case 'crash':
        ui.showMessage(json.player1.Name + ' and ' + json.player2.Name + ' had a crash');
        break;
      case 'kill':
        ui.showMessage(json.player1.Name + ' took out ' + json.player2.Name);
        break;
    }
  });
});
